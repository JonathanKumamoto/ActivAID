Templates

Many users find that the majority of the test programs they create follow the same format and use much of the same code.  Templates may be used to save time and reduce the chance of errors introduced from copying and pasting.  Templates are user-defined XML files that can be used for creating new test programs, group nodes and test nodes.  The following sections describe how to define and use template files in each of these situations:

Group Node Templates

Template File Overview

An ActivATE™ template file can be edited directly in the ActivATE™ IDE which provides XML color coding for easier editing.  It is recommended that you modify an existing template file rather than trying to create one from scratch.  Two template files are installed with ActivATE in the 'Templates' folder under the ActivATE installation directory -  "Blank.template" and "Sample.template".  The Blank.template file contains an empty set of XML tags, and the Sample.template file contains the XML tags populated with a template for a basic test program.   Both of these files can be modified to create your own template, but the XML tags should not be edited.  All template files need to have the ".template" file extension and must be located in the 'Templates' folder (usually located at "C:\Program Files\Racal Instruments\ActivATE .Net 3.x\Templates").  The XML tags are shown below:

            Blank.template file contents:

<?xml version="1.0" encoding="utf-8" ?> 
<!-- This is a comment -->
<CodeTemplate>
<NewTestProgram>
<!-- 
NewTestProgram Default parameters:  
	Test Program Name - $Name$
	Description - $Desc$
	Pre-Test Name - $PreTest$
	Post-Test Name - $PostTest$ 
-->
<Global>	
	<!-- Define any code to go in global space here (to be inserted at the top of the file) -->
</Global>
<PreTest>	
	<!-- Define your pre-test subroutine here -->
</PreTest>
<PostTest>	
	<!-- Define your post-test subroutine here -->
</PostTest>
</NewTestProgram>

<NewGroupNode>
<!-- 
NewGroupNode Default parameters:  
	Group Name - $Name$
	Description - $Desc$
	Pre-Group Name - $PreGroup$
	Post-Group Name - $PostGroup$ 
	Parent Node - $Parent$
-->
<Global>	
	<!-- Define any code to go in global space here (to be inserted immediately before the pre and/or post group subs) -->
</Global>
<PreGroup>	
	<!-- Define your pre-group subroutine here -->
</PreGroup>
<PostGroup>	
	<!-- Define your post-group subroutine here -->
</PostGroup>
</NewGroupNode>
<NewTestNode>
<!-- 
NewTestNode Default parameters:  
	Test Node Name - $Name$
	Description - $Desc$
	Function Name - $Function$
	Parent Node - $Parent$
-->
<Global>	
	<!-- Define any code to go in global space here (to be inserted immediately before the test function) -->
</Global>
<TestFunction>  
	<!-- Define your test function here -->
</TestFunction>
</NewTestNode>
</CodeTemplate>
