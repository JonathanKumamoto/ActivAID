Test Program Overview

All ActivATE test programs are comprised of a sequence file (.seq extension) and at least one test script file (.tpf extension).  The sequence file format is an Extensible Markup Language, or XML file and displayed graphically as a TreeView (much like a directory structure).  It contains information about the test program such as the test and group nodes, test properties, execution sequence and other user configurable settings.  This file is generated automatically by ActivATE, and in general, it should not be edited by the user.

The test script file contains the main VBScript, C#, or VB.Net code for the test program.  The outline for this test script is created automatically when a new test program is created and as new test and group nodes are added.  The user completes this file by adding the test code needed to implement each test. 

Optionally, for VBScript files, a globals file may be included (.gtf extension).  The globals file is also a VBScript file and is used to implement methods, properties and classes that may be used by multiple test programs.

These three files are named after the test program.  For example, a new test program created with the name "MyTest" would automatically create the files MyTest.seq, MyTest.tpf, and, if VBScsript and a globals file is to be used, MyTest.gtf.

Organizing a Test Program

Test Engineers should put some thought into designing a test program (sometimes called a test flow) and how the logical groups are organized.  For example, there should be a "Setup" or initialization group, a main test group, and a "Cleanup" or shutdown group.  This allows the test engineer to partition and validate the test program more logically.  Sections can easily be disabled (right-mouse-click on the node) or run separately.

  • Think about logical grouping

  • Don't try to do too much in one test. Each test node should be no longer than a page or page and half when viewed from the editor.  Break the test into relatively small, manageable parts.

  • Groups, as well as individual tests can be moved around (drag and drop) after the test flow is created.

  • As a general rule, the "Shutdown" group is used as a branch node incase there are catastrophic errors (see test sequence below.  The blue flag on the test node shows this node has the branch-on-fail flag set.

  • Keep the test names descriptive or add a description (shown as a blue comment).

  • As many sub-groups as needed may be used to add clarity and control to the test sequence.
