Node Drag-Drop

The group and test nodes in the Test Sequence View tree may be moved around within the same test program or they may be copied over from another test program.  This is accomplished by dragging and dropping the node to be moved or copied in the desired destination.  

Moving a node within the same test program

  1. Click on the node that is to be moved.  If a group node is selected, its entire subtree will be moved as well.
  2. Drag it to another node within the same Test Sequence View.
  3. Release the left mouse button to drop the node in its new location.
  4. The test script remains unchanged.

Copying a node from one test program to another

  1. Open both test programs in the IDE.
  2. Make each test program visible by selecting either "New Horizontal Tab Group" or "New Vertical Tab Group" from the Window menu.
  3. Click on the node that is to be copied.  If a group node is selected, its entire subtree will be copied as well.
  4. Drag it to another node in the other test program's Test Sequence view.
  5. Release the left mouse button to copy the node to its new location.
  6. The VBScript methods for the copied nodes are appended to the end of the destination test program's test script.  The test script of the source test program remains unchanged.  (See caution below.)

If a node is dragged onto a group node, it will be added as the last node in that group.  If a node is dragged onto a test node, it will be inserted directly above that node.  A group node cannot be dragged to any of its child nodes.  To cancel a drag-drop operation before the left mouse button is released, press the Escape key.  The root node cannot be moved or copied.

Two nodes with the same name cannot exist within the same group (i.e., at the same tree level).  Therefore, if you try to drag a node into a group that already contains a node with the same name, you will receive an error message and the drag-drop operation will not be completed.

Caution:  If a node is copied from one test program to another and the node's VBScript method has the same name as a previously existing method in the destination test program, the new node's method will still be copied to the test script.  This will result in the test script containing two methods with the same name.  This will not cause a script error when the test is run, but it is unknown which method the script engine will use.  It is the user's responsibility to ensure that each method has a unique name.
