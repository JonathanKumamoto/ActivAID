Test Program Manager: An Overview

 

The Test Program Manager GUI can be displayed by clicking on the Test Program Manager ICON in the Main Device Toolbar.

The Test Program Manager (TPManager) GUI is a simple user interface that can make calls directly into the TPMngr class. 

A more customized, application specific user interface can easily be developed using the TPManager class TPMngr to schedule test programs and display test results.  See the SystemController as an example.

The Test Program Manager module is designed to act as the "broker" between the front-end ActivATE.exe application and the back-end StationRuntime.exe application (see System Architecture)

The Test Program Manager (TP Manager) GUI below shows a number of test programs as loaded or running.  The TP Manager is capable of running separate and unique test programs in any of the available languages (C#, VBScript, VB.Net).

To Load and Run a test program from the Test Program Manager GUI:

 

 

  • Use the File Browser button on the TPMngr GUI to browse and select a test program.

 

 
  • Select any number TIUs(Test Interface Unit), sometimes called a test head as shown at right.  Use standard Microsoft multiple selection keys (CTRL-Mouse-Click, SHIFT-Mouse-Click).

 

  • Use the "Set File" button to "set" this file name in the select TIU positions.  You may set file names using all the same file or any multiple files.  They must be valid test programs.

 
   
  • Once selected and "Set", the file names will appear in the File Name column of the TP Manager GUI as shown at right.
 
  • Once a file name has  been set, the buttons shown at the right (Run, Pause, Stop, Load, Abort, Update) will only act upon CHECKED TIU items and only on checked items.  For example, if we check slots 3 and 4, but not slot 6, and then press LOAD, only slots 3 and 4 will be loaded with the indicated test program.

 

 

  • Likewise, we can selectively Run test programs using the same TIU check box and buttons.
If starting multiple slots, user can space the test start time between each slot by setting the Dwell Start. This value is in seconds and determines when to start the next slot testing after starting the previous slot (TIU) .   The default value is 2 seconds but may be set as long as 3600 seconds (that is, each slot can start 1 hour apart if need be).  In this example, Dwell Time is set to 20 seconds. 

Test Program Events

If a test program is executing from the TPMngr, the TPMngr will catch all sequencer events and pass them along to any object that subscribes to the events.  TMPngr events are:

// Bit 1 always set on a failure. Use combinations of bit weight.

public enum eTestStatus
{
    Passed =      0x00,
    Failed =      0x01,
    Stopped =     0x02,
    Running =     0x04,
    Failed_And_Running = 0x05, // failed but still running
    NotLoaded =   0x08,
    Paused =      0x10,
    ScriptError = 0x21,        // error + fail
    Loaded =      0x40,
    Aborted =     0x81,
    LoadFailed =  0x41,
    Message =     0xF0,        // General message
    ElapsedTime = 0xF4,        // ElapsedTime update
    StartTime =   0xF8,        // Start Time
    ScriptTrace = 0xFE         // Trace script tracelog ouput
    Unknown =     0x100,
}
