Using a null script name

A limitation of VBScript is that we cannot declare a class inside of an imported file when used as described above.  To work around this, you may declare your Import statement with a null, or empty, test script reference name.  In this case, the code inside the imported file is added to the main test script before being passed to the script engine instead of creating it as a separate object.  To use methods and variables declared outside of a class in the imported file, no object name is used.  If a class is declared inside the imported file, an instance of that class must be created before using its members (standard VBScript protocol).  

Example:  Repeat the previous example using a null script name.

  Option Explicit       ' MUST be declared on the first line
   
  Import "MathStuff.tpf", ""     
     .
     .
     .

  ' Call a function declared inside MathStuff.tpf
  Dim x, y, z
  x = 5 : y = 10
  z = Average(x, y)
  

Here, Average() is treated just as if it were declared locally in the main test script file.

 

Example:  Declare a class inside an import file.

  Option Explicit       ' MUST be declared on the first line
   
  Import "MathStuffClass.tpf", ""     
     .
     .
     .

  ' Instantiate an object of type Math and call one of its functions
  Dim mathObj
  Set mathObj = New Math
  Dim x, y, z
  x = 5 : y = 10
  z = mathObj.Average(x, y)
 

The Math class is declared in MathStuffClass.tpf as follows:

Class Math
' Returns the average of x and y
Function Average(x, y)
	Average = (x + y)/2
End Function
' Returns the sum of x and y
Function Sum(x, y)
	Sum = x + y
End Function
' Returns the difference between x and y
Function Difference(x, y)
	Difference = x - y
End Function
End Class
