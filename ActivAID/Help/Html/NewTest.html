Adding a new test

  1. Right click on the node to add a test node to.  Tests may ONLY be added to the root node or group nodes.
     
  2. Select "New Test" from the context menu.

   

  1. The New Test dialog appears as shown below.  Enter a name for the new test in the Test Name text box.

   

  1. Enter a description for this test if desired in the Description text box.
     
  2. Select the test type from the Test Type drop-down list.  The default value is Boolean.  For more information, see Test Types.
     
  3. If a Min/Max test type was selected, the Minimum and Maximum text boxes will be enabled.  Enter these values now if desired.  (They may also be edited later in the Property Grid or during runtime from the test program.)
     
  4. You may choose to select a node from the list of all available group and test nodes in the Branch On Fail Node drop-down list.  This designates the next node where execution will continue if this test fails during program execution.
     
  5. If you would like to create your test from an existing template, check the "Use Template" check box and select the template from the drop down list.  For information on creating templates, see the Templates help topic.
     
  6. The function name will automatically be filled in for you based on the test name you enter.  If you would like to change the default name, you may do so.  This function is the code that executes when this test is active during a test program run.
     
  7. Click OK to finish or Cancel to exit without creating a new test.
     
  8. The following code demonstrates the script that is auto-generated and displayed in the Script View window after the new test is created.  It is appended to the end of the existing test script.
'----------------------------------------------------
' Parent  : Group1
' Description: A sample test
'----------------------------------------------------
Function Test1()

	'  ToDo: Put your code here

	Test1 = false
End Function

 

Notes:

Two nodes with the same name cannot exist within the same group (i.e., at the same tree level).  Therefore, if you try to add a test node to a group that already contains a node with the same name, you will receive an error message and the new node will not be added.  (Node names are case-sensitive.)

If your new test node's VBScript function has the same name as a previously defined function, the following message box appears:

We are adding a new test node, Test1, to the test program.  This warning message appears because there is already a function named Test1 defined in the test script.

If you click "Yes", a new test function will not be created.  Instead, the new test node will execute the existing function when the test program is run.  It is OK to have two or more nodes call the same test function.

If you click "No", a new test function will be created, but a number will be appended to the end of the new function name to make it unique.  For example, if the function name is Test1, the names Test1_1, Test1_2, etc. will be tried until a unique name is found.

If you click "Cancel", the new node is not created.
