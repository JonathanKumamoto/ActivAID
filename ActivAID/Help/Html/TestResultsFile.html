Test Report File

The test results collected during each test program run may be saved to a file for later reference.  The properties and methods discussed here control the behavior of the test results file. 

When file output is enabled, a new test results file is written for each test program run on each TIU.   The file is saved to the path specified by ResultsFilePath and is named after the test program, TIU number, date and time.  For example, a test results file named MyTest_TIU5_01-17-06_14.01.53.pdf references a test program named MyTest that was run on TIU five on January 17, 2006 at 2:01:53pm.

It is also possible to create a "details" result file, which is used to record measurements made during a test.  The details files are saved in CSV (comma-separated values) format in the same location as the main test results file and are named after the main results file but with "_Details" appended (i.e., "MyTest_TIU5_01-17-06_14.01.53_Details.csv").  Results file output must be enabled for the details files to be written.  Use the LogMeasurement method to add data to the details file.

HeaderComment

Adds a user-defined comment to the header of the test results file.  You may wish to use this field to specify the serial number of the unit under test as was done in the sample test results report shown above.

Example VBScript call:  

  
  Results.HeaderComment = "S/N = ABC123XYZ"

 

LogMeasurement

void LogMeasurement( object min, object max, object actual, string units, string comment)

Adds a measurement to the details test result file. 

Example VBScript call:  


Dim dActual : dActual = DMM.ReadValue
Call Results.LogMeasurement(4, 5, dActual, "VDC", "DMM measurement")

 

ResultsFileEnable

If true, a test results file will be created and saved to the location specified by ResultsFilePath.  If false, the file will not be created.  This property is false by default and is only accessible from the properties page.

ResultsFilePath

The full path of the folder the test results file will be saved in if ResultsFileEnable is true.  By default, this path is set to C:\Temp\ActivATE\TestResults\.  This property may be set from the properties page or via the test script using the Results object.


Example VBScript call:  

  
  Results.FilePath = "C:\Temp\MyTestResults\"
