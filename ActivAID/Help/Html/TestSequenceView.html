Test Sequence View

The Test Sequence View is a tree view similar to the directory tree structure used in Microsoft Windows Explorer.  It determines the sequence of the test program's execution.  It minimally contains the root (top-level) test program node and may include any number of group and test nodes.  When a test program is run, execution begins with the root node and follows the tree nodes down in the order they appear.  If a PreTest method is specified for the root node, it will be the first method to execute.  For each subsequent node, execution will be determined according to the type of node.  For group nodes, the PreTest method (if specified) will execute first, followed by the test functions for each of the test nodes included in the group.  After all the group's child nodes have completed, the group's PostTest method is executed.  After all enabled nodes in the Test Sequence View have been executed, the PostTest method of the root node is run and the test program is complete.
