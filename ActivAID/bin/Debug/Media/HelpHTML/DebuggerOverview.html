Debugger Overview

The script debugger is an extremely useful tool for developing test programs.  It helps the user verify test program logic and determine the source of errors.  To use the debugger effectively, it is important to understand how to utilize breakpoints and the watch windows to your advantage.  

A breakpoint is a marker in the test program that signals the debugger to stop when execution reaches that point.  A breakpoint may be set on a group node, a test node, a line in the Script View, or a line in the Globals View.  When the debugger reaches a breakpoint and pauses, it is in a "break" state.  A breakpoint on a group node causes the program to break at the first line of executable code in the group's PreTest function, and a breakpoint on a test node causes the program to break at the first line of executable code in the test's function.  When the program stops at a breakpoint, the next line of code that will be executed is highlighted in yellow. 

The watch windows are used to "watch" the values of the test program's variables.  The Locals Watch Window watches only those variables that are local to the current function, and the Watch Window watches any variables the user requests it to.  Use the watch windows to verify that variables contain expected values. 

Once the test program reaches a breakpoint, you may examine variables in the watch windows, investigate the current state of the system by using the driver UI's, or step through the code line by line.  You may also add additional breakpoints or remove existing breakpoints before continuing the program.

Tip:  Once the test program starts execution in debug mode, it will stay in debug mode until either the program completes or is stopped by the user.  Therefore, clicking run to resume the program after stopping at a breakpoint will not prevent the program from breaking at subsequent breakpoints.

General steps for using the debugger:

  1. Set breakpoints in the code and/or the Test Sequence View.
  2. Run the debugger.
  3. After a breakpoint is reached, step through the code while examining variables in the watch windows.
  4. Click run to continue running in debug mode until either the next breakpoint or the end of the program, or click stop to end debug mode.
  5. If necessary, make changes to your code and re-run in debug mode.
