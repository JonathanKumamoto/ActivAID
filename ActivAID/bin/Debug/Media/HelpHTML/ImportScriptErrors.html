Script errors in an imported file

If a script error is generated in a VBScript file that is included with an Import statement, the error message will be displayed in the Script Trace Log window when the program is run.  To quickly navigate to the line where this error occurred, double-click on the error message in the Script Trace Log window.  The import file will be displayed in a new tab within the test program view, and the line with the error will be highlighted in yellow.  You may now make any necessary changes to correct the error, save the file and re-run the program.

Example:

A script error occurred in our imported file, MathStuff.tpf.  Double-click anywhere on the error message in the Script Trace Log box.

    

MathStuff.tpf is displayed in a new tab and the line with the error is highlighted.  Here, a semicolon at the end of line 4 is causing the error.  We simply remove the semicolon, save the file and re-run the test program.
