Modifying a Driver

The driver properties can be viewed and many can be edited directly from the properties page on the Configuration pane of the Outlook-style toolbar.  To access the properties for a driver, select that driver from the tree view and its properties will be displayed in the properties page.  To change a property that is editable, simply enter the new value in the properties page.

The properties for the DMM driver are displayed in the properties page here.


Device/Tool Properties

Property Name Description Read/Write
ClassName Full name of the driver class including the namespace and the name of the assembly file in the format "Namespace.DriverClass, AssemblyName". Read only
CLSID The Class ID, a universally unique identifier (UUID) that identifies a COM component.  This field will be empty ("") for .NET assemblies. Read only
Description A brief description of the device. Read/write
Enabled True if the driver object will be created for this device, otherwise false. Read/write
IsTool True if this assembly is a non-ActivATE™-specific instrument driver or a non-instrument "tool", such as a math or Excel module.  Read only
Name The object name of this device that will be used to reference it from the test program.  The name must be unique. Read/write
ObjectType The type of assembly, either .NET or COM. Read only
Scriptable True if the driver can be accessed from the test program, otherwise false. Read/write

Control Properties

Property Name Description Read/Write
ClassName Full name of the UI class including the namespace and the name of the assembly file in the format "Namespace.UIClass, AssemblyName". Read only
Description A brief description of the device.  This description will be displayed beneath the driver icon in the device toolbar. Read/write
Driver The name of the driver this UI is associated with. Read/write
Enabled True if the UI should be enabled and appear in the device toolbar. Read/write
Group The group name that this driver will appear under in the Outlook-style toolbar.  The default is "Devices". Read/write
Name The name of the UI object, typically the same as the associated driver object.  The name must be unique. Read/write

 

Important:  Committing Changes

After a driver's properties have been edited, the Reinitialize button on the toolbar will be highlighted in yellow as shown below.  The new driver settings will NOT be available until the environment has been reinitialized or until the application is restarted.  To reinitialize, make sure all test programs and driver UI's are closed and then click the Reinitialize button.

If at any time you wish to cancel your changes, click the Rollback button at the bottom of the properties page.  This will undo all changes that have been made since the last reinitialization.  The Rollback button is only enabled if there are uncommitted changes (until the system is reinitialized or until the application is shut down).
