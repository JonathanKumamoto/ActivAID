Adding a new group

Groups may be added to the test program to help section your tests into meaningful and manageable parts.  Test groups are analogous to directory folders on a computer disk.

  1. Right click on the node to add a group node to.  Groups may ONLY be added to the root node or other group nodes.
     
  2. Select "New Group" from the context menu.

   

  1. The New Group dialog appears as shown below.  Enter a name for the new group in the Group Name text box.

   

  1. Enter a description for this group if desired in the Description text box.
     
  2. If you would like to create your group from an existing template, check the "Use Template" check box and select the template from the drop down list.  For information on creating templates, see the Templates help topic.
     
  3. The Pre-Group and Post-Group subroutine names will automatically be filled in for you based on the group name you enter.  If you would like to change the default names, you may do so.  You may also choose to leave them blank.  The Pre-Group subroutine is executed before any tests in the group, and the Post-Group subroutine is executed after all other tests in the group have been completed.
     
  4. Click OK to finish or Cancel to exit without creating a new group.
     
  5. The following code demonstrates the script that is auto-generated and displayed in the Script View window after the new group is created.  It is appended to the end of the existing test script.
'----------------------------------------------------
' Parent  : Root
' Description: Pre Group Test: A group for my test
'----------------------------------------------------
Sub PreGroup1()
'  ToDo: Put your initialization code here
End Sub
'----------------------------------------------------
' Parent  : Root
' Description: Post Group Test:  
'----------------------------------------------------
Sub PostGroup1()
'  ToDo: Put your initialization code here
End Sub


Notes:

Two nodes with the same name cannot exist within the same group (i.e., at the same tree level).  Therefore, if you try to add a group node to a group that already contains a node with the same name, you will receive an error message and the new node will not be added.  (Node names are case-sensitive.)

If your new group node's VBScript methods have the same names as previously defined methods, the following message box appears:

If you click "Yes", new PreTest and PostTest subroutines will not be created.  Instead, the new group node will execute the existing subroutines when the test program is run.  It is OK to have two or more nodes call the same VBScript method.

If you click "No", new PreTest and PostTest subroutines will be created, but a number will be appended to the end of the new names to make them unique.  For example, if the node name is Group1, the names PreGroup1_1, PreGroup1_2, etc. will be tried until a unique name is found.

If you click "Cancel", the new node is not created.
