Test Types

Each test node must have a designated test type.  The test type gives the user more flexibility over how a pass or fail is determined for each test node.  The available test types are Boolean, Max, Min, MinMax and StringCompare.  Although a test type must be specified when a new test node is added to the program, it can be changed later through the Properties Page.  (See Test Node Properties)

Boolean Test Type

The default test type is Boolean.  A Boolean test returns either true or false where true means the test passed and false means the test failed.  When a Boolean test is added, the function template that is created for you in the test script will look similar to the following example:

Function Test1()

' ToDo: Put your code here

    Test1 = False
End Function

By default, the function returns false.  When you write the VBScript code for this test, you will want to return true or false depending on conditions that have or have not been met.

 

Max Test Type

A Max test returns a numeric value that is compared against the test node's Max value.  (The Min property is not applicable.)  If the return value is less than or equal to the Max value, the test passes.  Otherwise, the test fails.  When a Max test is added, the function template that is created for you in the test script will look similar to the following example:

Function Test2()

    ' ToDo: Put your code here

    Test2 = 0
End Function

By default, the function returns 0.  You may return any valid integer or floating-point number.  The Max value may be set when the test is created, in the Properties page, or from the VBScript test function.  See Test result properties of test nodes.

 

Min Test Type

A Min test returns a numeric value that is compared against the test node's Min value.  (The Max property is not applicable.)  If the return value is greater than or equal to the Min value, the test passes.  Otherwise, the test fails.  When a Min test is added, the function template that is created for you in the test script will look similar to the example shown above for the Max test type.

By default, the function returns 0.  You may return any valid integer or floating-point number.  The Min value may be set when the test is created, in the Properties page, or from the VBScript test function.  See Test result properties of test nodes.

 

Min/Max Test Type

A Min/Max test returns a numeric value that is compared against the test node's Min and Max values.  If the return value is greater than or equal to the Min value and less than or equal to the Max value, the test passes.  Otherwise, the test fails.  When a Min/Max test is added, the function template that is created for you in the test script will look similar to the example shown above for the Max test type.

By default, the function returns 0.  You may return any valid integer or floating-point number.  For example, a test might return a DMM reading which needs to lie within a certain range in order for the test to pass.  The Min and Max values may be set when the test is created, in the Properties page, or from the VBScript test function.  See Test result properties of test nodes.  

 

StringCompare Type

A StringCompare test returns a string value that is compared against the test node's ComparisonString property.  If the return value is equal to the ComparisonString, the test passes.  Otherwise, the test fails.  To make the comparison case-sensitive, set the node's CaseSensitive property to true (otherwise set to false).  When a StringCompare test is added, the function template that is created for you in the test script will look similar to the following example:

Function Test3()

    ' ToDo: Put your code here

    Test3 = ""
End Function

By default, the function returns an empty string.  You may return any string value.  The CaseSensitive and ComparisonString properties may be set when the test is created, in the Properties page, or from the VBScript test function.  See Test result properties of test nodes.
